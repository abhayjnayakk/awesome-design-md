# Choose A DESIGN.md Style

Use this guide when you are not sure which visual system to copy into your project.

## Fast Decision Matrix

| Product you are building | Start with | Add contrast from |
|--------------------------|------------|-------------------|
| AI agent SaaS | `vercel`, `cursor`, `voltagent` | `claude`, `notion` |
| AI media tool | `runwayml`, `elevenlabs` | `apple`, `theverge` |
| Developer API | `vercel`, `supabase`, `mintlify` | `linear.app`, `resend` |
| Analytics dashboard | `linear.app`, `sentry`, `clickhouse` | `notion`, `intercom` |
| Creator store | `shopify`, `airbnb`, `nike` | `starbucks`, `figma` |
| Fintech product | `stripe`, `wise`, `coinbase` | `mastercard`, `linear.app` |
| Research or writing app | `notion`, `wired`, `claude` | `linear.app`, `mintlify` |
| Premium hardware | `apple`, `tesla`, `spacex` | `bmw`, `nvidia` |

## Selection Rules

1. Pick one primary style for structure.
2. Pick one contrast style for personality.
3. Do not copy logos, names, copy, or exact hero layouts.
4. Convert references into a new `DESIGN.md` for your product.
