# DESIGN.md Style Gallery

A curated DESIGN.md style gallery and workflow for AI builders.

Most AI-generated apps look the same because the design prompt is vague: "modern SaaS UI," "clean dashboard," "make it beautiful." This repo helps you give coding agents a stronger visual brief using `DESIGN.md`: a plain-text design system file that agents can read before generating UI.

## What This Repo Is

This is a free, open-source starting point for:

- Finding high-quality `DESIGN.md` references.
- Choosing a style family for your product.
- Creating an original visual direction without cloning a famous brand.
- Giving Cursor, Claude Code, Copilot, or other AI coding agents better design context.

## Quick Start

1. Open [`style-taxonomy.md`](style-taxonomy.md).
2. Pick the product category closest to your app.
3. Review [`awesome-design-md-links.md`](awesome-design-md-links.md) for reference sources.
4. Try one of the starter examples in [`examples/`](examples/).
5. Use [`prompts/free-prompts.md`](prompts/free-prompts.md) with your coding agent.

## Free Starter Examples

This repo includes three original starter systems:

- [`examples/ai-devtool/DESIGN.md`](examples/ai-devtool/DESIGN.md)
- [`examples/creator-commerce/DESIGN.md`](examples/creator-commerce/DESIGN.md)
- [`examples/editorial-saas/DESIGN.md`](examples/editorial-saas/DESIGN.md)

They are intentionally generic and remixable. They are not clones of real brands.

## How To Use With An Agent

Copy a `DESIGN.md` into your project root, then prompt your agent:

```text
Before editing UI, read DESIGN.md and treat it as the visual source of truth. Use its colors, typography, spacing, component rules, and do/don't guidance. If something is missing, propose a DESIGN.md update before inventing new styles.
```

## More Useful Than A Raw List

A raw collection gives you inspiration. This repo adds:
